# Security Policy

## Reporting a vulnerability

Please do not open public issues for security-sensitive reports.

If you believe you found a vulnerability, contact Fernando Paladini privately through GitHub or by email at fnpaladini@gmail.com and include:

- A clear description of the issue.
- Steps to reproduce it.
- The affected browser and extension version.
- Any proof of concept that helps validate the report.

## Scope

Relevant security reports include, for example:

- Data leakage across courses.
- Unsafe export behavior.
- Backup and restore issues that could corrupt user data.
- Privilege misuse inside the extension.

## Response

Fernando Paladini will review reports and aim to confirm, prioritize, and address valid issues as quickly as practical.
