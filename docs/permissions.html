Permissions justification

This page explains why the extension requests each Chrome permission.

storage

Used to store captured course content, user settings, backup metadata, and recovery snapshots locally inside the browser.

activeTab

Used to interact with the current tab when the user clicks the extension on a Moodle page. This keeps access limited to the page the user is actively viewing instead of requiring broad permanent host access.

scripting

Used to inject the extraction logic into the current tab only when the user requests a capture. This lets the extension read the visible Moodle content and save it locally without automatic page-wide injection.

downloads

Used to export captured data as Markdown files and to export JSON backups when the user clicks export actions.

Why the extension injects on pages

The extension injects only on the current page after a user action in the popup. It then checks whether the page matches Moodle structure and extracts content only from that page.

Data handling summary

  • Data is processed locally.
  • Data is not sold.
  • Data is not transferred to a remote backend by default.
  • Files are only downloaded when the user explicitly requests export.
