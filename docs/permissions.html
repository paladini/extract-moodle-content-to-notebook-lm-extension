Permissions justification

This page explains why the extension requests each Chrome permission.

storage

Used to store captured course content, user settings, backup metadata, recovery snapshots, and capture state locally inside the browser.

activeTab

Used to interact with the current tab when the user actively uses the extension. This helps the extension work with the page the user is viewing without requiring broader permanent host access in the popup flow.

scripting

Used to coordinate extension logic on the current page and support content extraction workflows in Chromium environments that rely on extension scripting APIs.

downloads

Used to export captured data as Markdown files and to export JSON backups when the user clicks export actions.

Why the extension injects on pages

The content script runs on pages in order to detect Moodle pages and extract content only when the page matches the expected Moodle structure. The user controls when capture is active.

Data handling summary

  • Data is processed locally.
  • Data is not sold.
  • Data is not transferred to a remote backend by default.
  • Files are only downloaded when the user explicitly requests export.
